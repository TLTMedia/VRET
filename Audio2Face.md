 NVIDIA Audio2Face (A2F) has evolved from its original 46-blendshape template to fully supporting the ARKit 52 blendshape standard, and you can export this data as a JSON file.This workflow is common for developers who want to drive custom rigs in engines like Unity or Unreal without using the heavier USD format.How to Achieve ThisTo get a 52-blendshape JSON, you need to use the Blendshape Solve workflow rather than the basic Character Transfer.Use the ARKit Solver: In the Audio2Face UI, navigate to the Blendshape Solve tab. You should see a sample scene or preset called male_bs_arkit.usd or mark_solved_arkit.usd. This uses a mapping designed specifically for the 52 ARKit shapes.Character Setup: If you are using a custom character, you must first perform a Character Transfer to map your mesh to the A2F "Mark" model. Once transferred, you use the Blendshape Solver to "solve" those movements into ARKit-compatible weights.Export to JSON:In the Data Conversion or Export panel, look for the "Export Weights" option.Select JSON as the output format.Ensure your solver is set to the ARKit preset. The resulting JSON will contain a list of the 52 ARKit names (e.g., eyeBlinkLeft, jawOpen, mouthPucker) with their corresponding weight values for every frame.Important ConsiderationsFeatureDetailsNaming ConventionA2F uses the standard ARKit naming (e.g., eyeLookInRight). If your target rig uses different names, you may need a simple Python script to rename the keys in the JSON.Headless/APIIf you are automating this, the A2F REST API supports the ExportBlendshapes command, which can generate these JSON files programmatically.Custom MeshesFor the best results, your custom mesh should have all 52 ARKit blendshapes sculpted. If it doesn't, A2F can "generate" them for you using the Blendshape Generation tool, though manual cleanup is often required for high-fidelity characters.
